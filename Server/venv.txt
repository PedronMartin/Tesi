VENV (Ambiente Virtuale Python)

Un ambiente virtuale crea una copia isolata di Python e delle sue librerie solo per il progetto corrente.

PerchÃ© Usarlo
    Isolamento: Permette di installare librerie complesse (come OSMnx e Flask) senza interferire o danneggiare le librerie di sistema del tuo computer (risolvendo l'errore externally-managed-environment).

    Conflitti: Evita che versioni diverse di una libreria (es. Progetto A richiede Flask v2, Progetto B richiede Flask v3) entrino in conflitto.

Uso (Attivazione e Disattivazione)
Attiva con source venv/bin/activate	---> Il terminale mostra (venv) all'inizio.
Disattiva con deactivate	---> (venv) scompare dal terminale.

Attivalo sempre quando devi installare pacchetti o eseguire il tuo server Flask.
